Entonces, el tipo de dato ideal para la informacion financiera es un json, donde el json sera:
{ nameMethod: {
    accountReference: ,
    idAccount: ,
  }
}

Donde el nombre del metodo tiene que ser un enumerado (nombres de los distintos bancos, metodos electronicos y criptomonedas),
la referencia de la cuenta un numero o un string (numero de cuenta de banco o criptomoneda, email u otro dato para el medio electron
ico alternativo, como paypal, zelle, etc) y el id de la cuenta sera opcional, entonces, sera el numero de rif o cedula del 
la cuenta bancaria del comercio, etc.
Btw, el tipo de dato sera jsonb, ya que este me permitira ejecutar operaciones en la base de datos, de consulta, actualizacion,
etc. Y tambien, sera un array de jsonb.

El problema es el hecho de como hacer constantes el nombre del metodo de pago y las propiedades, quiero decir,
como los metodos y los atributos seran estaticos (banesco, bitcoin, ... y numero de referencia, id) entonces quiero que
las propiedades esten ya definidas para poder insertar los valores. Bueno... no perdere mas tiempo con eso. El
enfoque que usare ahora mismo sera el siguiente:
